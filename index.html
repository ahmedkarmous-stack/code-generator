Comment fonctionne un Code QR ?

Qu'est-ce qu'un Code QR ?

Un Code QR (Quick Response Code) est une matrice de données bidimensionnelle composée de modules noirs et blancs. Inventé en 1994 par Masahiro Hara chez Denso Wave, il peut stocker jusqu'à 7089 caractères numériques ou 4296 caractères alphanumériques.

Contrairement aux codes-barres linéaires, les codes QR encodent l'information dans deux dimensions, permettant une densité de données bien supérieure et une lecture omnidirectionnelle.

Structure anatomique d'un Code QR

◰
Finder Patterns

3 carrés de 7x7 modules dans les coins pour détecter la position et l'orientation

◫
Alignment Pattern

Carrés de 5x5 modules pour corriger la distorsion perspective

┄
Timing Patterns

Lignes alternées noir/blanc définissant la grille de coordonnées

▦
Format Information

15 bits encodant le niveau de correction et le masque appliqué

◧
Version Information

18 bits (versions 7+) indiquant la taille de la matrice

▤
Data & Error Correction

Zone principale contenant les données encodées et codes Reed-Solomon

Processus technique de génération

1

Analyse et choix du mode d'encodage

Le contenu est analysé pour déterminer le mode optimal : Numérique (0-9), Alphanumérique (0-9, A-Z, espace, $%*+-./), Byte (ISO 8859-1/UTF-8), ou Kanji.

// Indicateurs de mode (4 bits) Numérique: 0001 // 10 bits pour 3 chiffres Alphanumérique: 0010 // 11 bits pour 2 caractères Byte: 0100 // 8 bits par caractère Kanji: 1000 // 13 bits par caractère
2

Encodage des données en binaire

Les données sont converties en flux binaire selon le mode. Pour le mode alphanumérique, chaque paire de caractères est convertie : (valeur1 × 45) + valeur2 en 11 bits.

// Exemple: encoder "AB" "A" = 10, "B" = 11 (10 × 45) + 11 = 461 461 → 00111001101 (11 bits)
3

Construction du message binaire

Le flux final combine : indicateur de mode + compteur de caractères + données + terminateur (0000) + padding jusqu'à la capacité.

// Structure du message [Mode][Count][Data...][Terminator][Padding] 4b 9-16b variable 4b 11101100 00010001...
4

Génération des codes de correction d'erreurs (Reed-Solomon)

L'algorithme Reed-Solomon génère des codewords de correction permettant de récupérer les données même si une partie du QR est endommagée. Les données sont divisées en blocs, chacun recevant ses propres codewords EC.

Niveau Récupération Overhead
L (Low) ~7% des codewords ~20%
M (Medium) ~15% des codewords ~38%
Q (Quartile) ~25% des codewords ~55%
H (High) ~30% des codewords ~65%
5

Entrelacement des données

Les codewords de données et de correction sont entrelacés pour distribuer les erreurs potentielles. Si un bloc est endommagé, les erreurs sont réparties sur plusieurs blocs logiques.

// Entrelacement de 2 blocs Bloc 1: D1, D2, D3 | EC1, EC2 Bloc 2: D4, D5, D6 | EC3, EC4 Final: D1, D4, D2, D5, D3, D6, EC1, EC3, EC2, EC4
6

Placement dans la matrice

Les bits sont placés dans la matrice en suivant un motif en zigzag ascendant/descendant, en évitant les zones réservées (finder patterns, timing patterns, etc.).

// Direction de placement des modules ↑ ↑ Colonnes traitées par paires │ │ de droite à gauche ┌─┴─┴─┐ │ 2 1 │ ←── Commence en bas à droite │ 4 3 │ Remonte en zigzag │ 6 5 │ Puis redescend └─────┘
7

Application du masque

8 motifs de masque sont testés pour équilibrer la distribution noir/blanc et éviter les patterns problématiques. Le masque avec le score de pénalité le plus bas est choisi.

// Formules des 8 masques (appliqués avec XOR) 0: (row + col) mod 2 == 0 1: row mod 2 == 0 2: col mod 3 == 0 3: (row + col) mod 3 == 0 4: (row/2 + col/3) mod 2 == 0 5: (row×col) mod 2 + (row×col) mod 3 == 0 6: ((row×col) mod 2 + (row×col) mod 3) mod 2 == 0 7: ((row+col) mod 2 + (row×col) mod 3) mod 2 == 0
8

Ajout des informations de format et version

Les 15 bits de format (niveau EC + masque) sont encodés avec BCH et placés autour des finder patterns. Pour les versions 7+, 18 bits de version sont ajoutés.

// Format Information (15 bits) [EC Level][Mask Pattern][BCH Error Correction] 2b 3b 10b // XOR avec masque fixe pour éviter les patterns uniformes mask = 101010000010010

Versions et capacités

Il existe 40 versions de QR codes, de 21×21 modules (version 1) à 177×177 modules (version 40). Chaque version supérieure ajoute 4 modules par côté.

Version Taille Capacité max (L) Capacité max (H)
1 21×21 25 alphanum 10 alphanum
10 57×57 395 alphanum 174 alphanum
25 117×117 1853 alphanum 822 alphanum
40 177×177 4296 alphanum 1852 alphanum

Algorithme Reed-Solomon simplifié

La correction d'erreurs utilise les mathématiques des corps de Galois GF(2⁸). Les données sont traitées comme coefficients d'un polynôme, divisé par un polynôme générateur.

// Génération des codewords EC function generateEC(data, ecCount): // Polynôme générateur de degré ecCount generator = buildGenerator(ecCount) // Division polynomiale dans GF(2^8) for i in data: coef = data[i] XOR result[0] result.shift() for j in generator: result[j] XOR= gfMul(generator[j], coef) return result // Codewords de correction
